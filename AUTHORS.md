## Contributing

We welcome and appreciate contributions from the community! If you'd like to contribute to DiComics, please follow these guidelines:

### How to Contribute

1. **Fork the Repository**: Create a personal fork of the project
2. **Create a Feature Branch**: Use `git checkout -b feature/your-feature-name`
3. **Make Your Changes**: Implement your feature or fix
4. **Write Tests**: Ensure your changes are tested
5. **Commit with Clear Messages**: Use descriptive commit messages
6. **Push to Your Fork**: `git push origin feature/your-feature-name`
7. **Submit a Pull Request**: Open a PR with a detailed description

### Contribution Areas

We're looking for help with:

- **Bug Fixes**: Help identify and fix issues
- **Features**: Implement new features from the roadmap
- **Documentation**: Improve README, docs, and code comments
- **Tests**: Expand test coverage
- **Performance**: Optimize existing code
- **UI/UX**: Design improvements and refinements
- **Localization**: Add language support
- **Packaging**: Help with distribution and releases

### Code of Conduct

Please be respectful and constructive in all interactions. We're building a welcoming community for all contributors.

### Development Setup

See the [README.md](README.md) for setup instructions.

### Pull Request Process

1. Update documentation if needed
2. Add tests for new features
3. Ensure all tests pass
4. Update CHANGELOG.md with your changes
5. Provide a clear description of what your PR does

### Reporting Issues

- Use GitHub Issues for bug reports
- Include steps to reproduce
- Provide environment details (OS, Node version, etc.)
- Attach screenshots if relevant
